# Contributing to PathAI

Thank you for considering contributing! We welcome contributions that help improve PathAI for personal and educational use. This document explains the process and important legal notes about contributions.

## How to contribute

1. Open an issue to discuss major changes or features before implementing them.
2. Fork the repository and create a branch for your work: `git checkout -b feature/your-feature`.
3. Write clear, focused commits and keep PRs small when possible.
4. Add tests for any new functionality and ensure existing tests pass.
5. Follow the project's coding style and run `npm run format` before opening a PR.
6. Open a Pull Request describing the change, referencing related issues, and provide screenshots or notes as needed.

## Review process

- PRs will be reviewed by maintainers. Expect feedback and be prepared to make adjustments.
- Once approved, a maintainer will merge the PR.

## License and Copyright (Important)

This statement is intended to make the project's licensing and ownership expectations clear. It is not a substitute for a formal contributor license agreement or other legal document; for larger or commercial contributions, please contact the maintainers to discuss a formal agreement.

## Code of Conduct

Be respectful and professional. Harassment, hate speech, or abusive behavior will not be tolerated.

## Reporting Security Issues

If you discover a security vulnerability, please do not open a public issue. Instead, contact the maintainers privately using the contact information in the repository.
